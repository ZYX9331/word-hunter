# 单词猎手 Word Hunter 🔍

**一款以 CET-4 词汇记忆为目标、基于主动召回机制的单词搜索游戏。**

> **[在线游玩 →](https://zyx9331.github.io/word-hunter/)**

---

## 关于本项目

单词猎手是一个运行在浏览器中的单词搜索游戏，旨在让 CET-4 词汇复习变得更高效、更有趣。无需安装、无需注册、无广告，无论电脑、平板还是手机，打开链接即可使用。

整个项目是一个零依赖的单 HTML 文件，轻量、便携、可直接部署。它既是我个人的编程练习项目，也是本篇英语课程作业中关于"词汇学习策略"的实践载体。

---

## 问题背景

传统的词汇学习往往依赖死记硬背——反复阅读词表、翻看单词卡、机械默写。这类方法虽然能在短期内帮助识别单词，但长期记忆效果并不理想。

认知心理学研究表明，**主动召回（Active Recall）**——即主动从记忆中提取信息的过程——其学习效率远高于被动复习。然而，大多数主动召回工具（如间隔重复 App、听写测验）在长时间使用后容易让人感到枯燥。

单词猎手将主动召回融入游戏机制，让记忆成为游戏过程的自然产物，而非枯燥的任务本身。

---

## 游戏机制

一个 10×10 的字母矩阵中，横向或纵向隐藏了最多 16 个英语单词。玩家需要在 **3 分钟** 内尽可能多地找出它们。

核心机制在于：**在矩阵中找到单词只是第一步——你还必须在输入框中写出它的中文释义才能得分。** 这个"识别 → 召回"的两步过程，远比单纯识别单词更能强化记忆。

### 操作流程

1. 扫描字母矩阵，发现一个英语单词（例如找到 `heart`）
2. 在底部输入框中写出它的中文含义（如 `心脏` 或 `心灵`）
3. 按回车提交——答对时格子变绿 ✓，计 1 分

## 词库说明

游戏内置了从《2026年6月英语四级1500核心词》PDF 中提取的 **1640 个高频 CET-4 词汇**，每个词均配有中文释义。

同时支持上传自定义 CSV 词库，解析器兼容以下格式：

**简易格式：**
```csv
单词,释义1|释义2|释义3
book,书|书本|书籍
apple,苹果|苹果树
```

**标准 CET-4 格式（含词性标记）：**
```csv
abandon,v. 放弃，抛弃
ability,n. 才能，能力
```

解析器会自动去除 `v.`、`n.`、`adj.` 等词性前缀，仅保留中文字符作为有效答案。

## 学习科学基础

本游戏的设计基于以下三个学习原理：

### 1. 主动召回（Active Recall）
与单纯阅读词表不同，玩家必须主动回忆单词的含义。这一提取过程能强化神经通路，显著提升长期记忆效果。

### 2. 情境化学习（Contextual Learning）
单词被放置在字母矩阵中，玩家通过识别字母序列来发现单词，而非依赖词表上的固定位置。这种模式识别训练有助于在实际阅读中更快地辨认词汇。

### 3. 间隔提示（Spaced Hinting）
当玩家长时间没有操作时，游戏会自动以金色闪烁提示某个未找到单词的字母位置。这种温和的引导既能避免卡关挫败感，又保留了你主动回忆的努力。提示频率可通过难度滑块调节。

## 技术实现

- **技术栈**：纯 HTML + CSS + JavaScript，无框架、无构建工具、无 npm
- **单词放置算法**：回溯算法，长词优先，每词最多尝试 400 次
- **布局方案**：CSS Grid 配合响应式格子尺寸
- **移动端键盘适配**：利用 `visualViewport` API 将输入框固定在键盘上方
- **文件体积**：单文件 `index.html`，约 800 行
- **字体**：Google Fonts（Righteous、Space Mono、Noto Sans SC）

### 本地运行

```bash
npx serve .
# 或
python -m http.server 8000
```

然后访问 `http://localhost:8000`。（自动加载 CSV 词库需要 HTTP 服务器环境，浏览器会阻止 `file://` 协议下的 `fetch` 请求。）
