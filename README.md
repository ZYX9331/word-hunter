# 🔍 单词猎手 Word Hunter

> *一款用来让背 CET-4 单词变得好玩一点的网页版单词搜索游戏。*

**在线游玩 → [zyx9331.github.io/word-hunter](https://zyx9331.github.io/word-hunter/)**

---

## 为什么做这个

盯着单词表死记硬背效率很低。单词搜索游戏迫使你先**辨认**单词拼写，再**回忆**它的含义——这是认知科学里所说的"主动提取"（Active Recall），比被动重读的记忆效果好得多。

这个项目想验证一件事：一个零依赖、零构建工具的单 HTML 文件，能不能做出足够流畅的体验，真正用于日常背单词？

结论是可以。它能跑在浏览器里、手机桌面上，甚至 U 盘里。

---

## 玩法说明

10×10 的字母矩阵中横向或纵向隐藏着最多 **16 个英语单词**，你有 **3 分钟**尽可能多地找出它们。

**核心机制：** 光眼睛看到单词还不够，你必须在输入框里填写它的**中文释义**才能得分。这一步把"认识这个单词"和"真的记住它的意思"区分开来，正是四六级考试考查的核心能力。

```
第一步  扫描字母格，找到一个单词（比如发现了 "heart"）
第二步  在底部输入框输入中文含义（比如输入 "心脏" 或 "心灵"）
第三步  回车提交，答对的格子变绿 ✓，计 1 分
```

### 操作说明

| 界面元素 | 功能 |
|---|---|
| ⏸ / ▶ | 暂停 / 继续倒计时 |
| 拖动格子 | 将格子标记为珊瑚红色，方便临时做标注 |
| 难度滑块 | 设置闲置提示间隔：简单 5 秒 / 中等 10 秒 / 困难 20 秒 |
| 📂 上传卡片 | 导入自定义 CSV 词库 |

### 闲置提示

如果你一段时间没有任何操作，游戏会自动把某个未找到单词的字母闪烁**金色**——只是轻轻提示方向，不会直接告诉你答案。提示频率由难度滑块控制。

### 同义词匹配

答案检查有一定容错性。`书`、`书本`、`书籍` 都能匹配同一个单词；输入内容与存储释义存在包含关系时也视为正确。

## 词库说明

游戏内置 29 个常用词作为默认词库，在 HTTP 环境下会自动加载同目录的 CET-4 高频词库（1500）（`cet4-high-frequency-words.csv`）。

### 自定义 CSV 格式

上传 CSV 文件即可替换词库，游戏会立刻用新词库重新开始：

```csv
单词,释义1|释义2|释义3
book,书|书本|书籍
apple,苹果|苹果树
```

也兼容标准 CET-4 格式（含词性前缀），解析时会自动去掉 `v.`、`n.`、`adj.` 等标记，只保留中文作为有效答案：

```csv
abandon,v. 放弃，抛弃
ability,n. 能力，才能
```

## 技术实现

整个游戏是一个**单文件 `index.html`**（约 28 KB，877 行），无 npm、无打包工具、无框架。

| 方面 | 实现方式 |
|---|---|
| 单词放置 | 回溯算法，每个单词最多尝试 400 次，长单词优先放置 |
| 界面渲染 | 纯 DOM + CSS Grid + CSS 自定义属性 |
| 字体 | Google Fonts（Righteous、Space Mono、Noto Sans SC）|
| 移动端键盘适配 | `visualViewport` API，键盘弹出时输入框自动跟随 |
| 离线使用 | 支持添加到手机桌面作为 PWA 应用使用 |

### 本地运行

CSV 自动加载需要 HTTP 服务（浏览器在 `file://` 协议下会阻止 `fetch()`）：

```bash
# Node.js
npx serve .

# 或 Python
python -m http.server 8000
```

然后访问 `http://localhost:8000`。
